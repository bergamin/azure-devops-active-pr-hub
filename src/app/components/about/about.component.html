Active Pull Requests Hub - {{version}}

Esta aplicação visa facilitar o gerenciamento dos pull requests ativos no Azure DevOps.

Configuração

Preencher o PAT com um token gerado no Azure DevOps. Pode ser um token só com permissões de leitura. Por motivos de segurança, o PAT salvo não é apresentado no campo após o salvamento. Ao alterar outras configurações mantendo o PAT em branco, irá manter o PAT que já estava salvo anteriormente sem alterações.

A lista de repositórios aceita tanto o nome do repositório (e.g. ws-exemplo), quanto o id do repositório (e.g. 9fd49287-ee78-4f67-bfe5-d6a3b914805e). Desta forma, pode-se também utilizar neste filtro os repositórios da forma como são salvos no plugin Pull Request Manager Hub do Azure DevOps. Esta lista é separada por vírgulas e é sanitizada no momento do salvamento, removendo caracteres brancos e " que pode estar vindo de backup do plugin do Azure DevOps.

Caso o filtro de repositórios não for preenchido, serão mostrados todos os PRs abertos.

Pode-se exportar as configurações para um arquivo json para utilizar como backup. Porém, a exportação não salvará e a importação não substituirá o PAT.

Caso o avatar do autor dos PRs não for apresentado, isso indica que o usuário não está logado no Azure DevOps. Para esta funcionalidade funcionar corretamente, deve-se utilizar o mesmo navegador onde se trabalha com o Azure DevOps.
